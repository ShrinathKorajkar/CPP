common Markdown syntax elements:

1. Headings: 
   # Heading 1
   ## Heading 2
   ### Heading 3
 
2. Text Formatting:
   - Bold: **bold text**
   - Italic: *italicized text*
   - Strikethrough: ~~strikethrough text~~
   - Inline Code: `` `inline code` ``

3. Lists:
   - Unordered List: Use `-` or `*` to create unordered lists. 
   - Ordered List: Use numbers with period to create ordered lists. 

4. Links:
   [link text](URL) => [OpenAI](https://openai.com)

5. Images:
   ![alt text](image URL) => ![Logo](https://example.com/logo.png)

6. Blockquotes:
   > This is a blockquote.
   > It can span multiple lines.

7. Code Blocks:
   Use triple backticks (```) to create a code block.
   Specify the language after the opening backticks. 
   
   ```python
   def hello_world():
       print("Hello, World!")
   ```

8. Tables: 
   Use pipes (`|`) to create tables.
   
   | Column 1 | Column 2 |
   |----------|----------|
   | Data 1   | Data 2   |
   | Data 3   | Data 4   |
   






# Naming Convention
    1. Branch : 
	a. prefix - 
	    feature/ 
	    bugfix/ 
	    hotfix/ 
 	    release/ 
	    experiment/ 
	    chore/update-dependencies-or-routine-task-maintainace
	    refactor/refactor-without-adding-feature-fixing-bugs
	    test/
	    docs/update-readme
	    ci/setup-ci
	b. prefix/JiraId-name (lowercase and descriptive)

   2. Commit
	a. Subject (50 characters or less)
	   Blank line
	   Body (if needed, 72 characters per line) (use - or * to add points)
	b. ticket reference at beginning
